## Quick Start
## Usage
1. Login into your github account.
2. Set up your configuration & upload it to the system.
	- bits / inst
	- inst format
3. write your asm code in the editor.
4. Select the output format you prefer: [**binary**, **memory**]

## Formats
#### INST Rules
**Rules** defines when an instruction if following a valid configuration format.
A **valid configuration format** is one that matches one of the following patterns.

**Instruction**: /^([a-z0-9]+)/i<br/>
**Constant**: /^#([a-z])/i<br/>
**Register**: /^r([a-zA-Z]+)/<br/>
**Address**: /^\[r([a-zA-Z0-9]+)\]/<br/>
**Number**: /^([0-9]+)/<br/>
**Range**: /^([0-9]+\:[0-9]+)/<br/>
**Semicolon**: /^;/<br/>


If you think that a pattern does not work correctly try to validate it in [regex101](https://regex101.com/), if you see that the pattern is not being detected there leave an issue explaining the problem detailing the steps followed and data provided.

#### INST Format
The instruction format consists mainly of 2 parts: **[pattern];[ranges];**
**Patterns:** Defines the valid language for writing the instructions.
```
mov #K rd;		-- [inst] [cte] [reg]
add ra rb rd;	-- [inst] [reg] [reg] [reg]
```

**Ranges:** Defines the position where the field of the pattern read will be written.
```
0 32:32 20:5 4:0;		-- [co] [r_co] [r_k] [r_rd]
1 32:32 14:10 9:5 4:0;  -- [co] [r_co] [r_ra] [r_rb] [r_rd]
```

#### Complete INTS format
```
mov #K rd; 0 32:32 20:5 4:0;
```

## Example
#### Code
```
mov #30000 r1;
mov #30000 r2;
mov #30000 r3;
mov  #8000 r4;
mov     #0 r5;
mov    #90 r6;
mov     #5 r7;
add r1 r2 r0;
add r3 r0 r0;
add r4 r0 r0;
add r5 r0 r0;
add r6 r0 r0;
add r7 r0 r0;
```

#### Configuration
```
mov #K rd; 0 22:22 21:5 4:0;
add ra rb rd; 1 22:22 15:10 9:5 4:0;
```
```
INST: 22 bits
```
